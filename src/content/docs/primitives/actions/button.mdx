## Installation

```bash
npm install @milanlaukens/mmui
```

## Usage

```astro
---
import { Button } from '@milanlaukens/mmui';
---
<div class="not-content">

<Button label="Click me" />
```

## Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `label` | `string` | — | Button text |
| `variant` | `"default"` \| `"primary"` \| `"warning"` \| `"danger"` \| `"subtle"` \| `"link"` | `"default"` | Visual style |
| `size` | `"sm"` \| `"md"` \| `"lg"` | `"md"` | Size scale |
| `spacing` | `"default"` \| `"compact"` \| `"none"` | `"default"` | Inner padding |
| `disabled` | `boolean` | `false` | Disables the button |
| `selected` | `boolean` | `false` | Marks the button as selected |
| `loading` | `boolean` | `false` | Shows a spinner and blocks clicks |
| `fullWidth` | `boolean` | `false` | Stretches to 100% of its container |
| `href` | `string` | — | Renders as an `<a>` tag instead of `<button>` |
| `iconBefore` | `string` | — | Raw SVG string rendered before the label |
| `iconAfter` | `string` | — | Raw SVG string rendered after the label |

## Notes

<Aside type="tip">
  When `href` is provided, the button renders as an `<a>` tag instead of a `<button>`. The `disabled` prop still works — it adds `aria-disabled` and a `not-allowed` cursor, but since `<a>` tags can't be natively disabled, clicks are blocked via CSS `pointer-events: none`.
</Aside>

<Aside type="caution">
  The `iconBefore` and `iconAfter` props accept raw SVG strings. Make sure your SVG uses `currentColor` for stroke and fill so the icon color adapts automatically to the button variant.
</Aside>
